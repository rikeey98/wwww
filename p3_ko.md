# SoC RTL 검증을 위한 AI 에이전트 기반 사양서 매칭 시스템

## 초록

SoC RTL 검증 워크플로는 설계 사양서와 검증 테스트 케이스 간의 매칭 프로세스가 수동으로 수행되어 심각한 효율성 문제에 직면하고 있다. 본 논문은 RTL 설계 사양서와 검증 테스트 사양서를 자동으로 매칭하고, 매칭 결과를 바탕으로 기존 담당자 정보를 활용한 자동 알림 시스템을 제공하는 AI 에이전트 기반 통합 솔루션을 제시한다. 제안 시스템은 설계 의도와 검증 목표를 지능적으로 연관시키는 사양서 매칭 에이전트와 매칭된 정보의 담당자에게 자동으로 알림을 전송하는 통합 워크플로를 포함한다. 실제 SoC 프로젝트 데이터를 활용한 실험에서 완벽 매칭 가능 데이터(7만건)에서 98.5%, 부분 매칭 가능 데이터(5만건)에서 87.3%, 매칭 매우 어려운 데이터(3만건)에서 62.1%의 정확도를 달성하였으며, 전체 평균 84%의 시간 단축 효과와 함께 즉시 담당자 알림을 통한 신속한 문제 해결 체계를 구축하였다.

**키워드:** AI 에이전트, SoC 검증, 사양서 매칭, 자동화 알림

## 1. 서론

현대 SoC RTL 검증은 기능적 사양서를 작성하는 설계 팀과 해당 테스트 케이스를 개발하는 검증 팀 간의 복잡한 조정을 포함한다. 설계 사양서와 검증 테스트 케이스 간의 연결을 설정하는 과정은 현재 대부분 수동으로 수행되며, 이로 인해 상당한 시간 소모와 오류 가능성이 발생한다.

특히 대규모 SoC 프로젝트에서는 하나의 항목에 대해서도 검증자마다 서로 다른 검증 환경을 구성하여 테스트를 수행하기 때문에, 실패 발생 시 원래 설계 사양서를 찾고 매칭하는 과정이 매우 복잡하고 시간 소모적이다. 엔지니어들은 일반적으로 적절한 사양서를 찾기 위해 2-8시간을 소요하며, 복잡한 케이스의 경우 며칠이 걸리기도 한다.

또한 적절한 담당자를 찾아 문제를 전달하는 과정에서도 추가적인 지연이 발생하며, 이는 전체 검증 프로세스의 효율성을 크게 저하시킨다.

본 논문은 이러한 사양서 매칭 프로세스를 자동화하고, 매칭 결과를 바탕으로 관련 담당자에게 즉시 알림을 전송하는 통합 AI 에이전트 기반 시스템을 제안한다.

## 2. 시스템 아키텍처

### 2.1 전체 프레임워크

제안 시스템은 RTL 설계 사양서, 검증 테스트 사양서, 담당자 정보, 과거 매칭 이력을 포함하는 통합 데이터 환경 내에서 작동한다. 사양서 매칭 에이전트가 핵심 매칭 기능을 수행하고, 자동 알림 시스템이 결과를 관련 담당자에게 전달한다.

```
[RTL 설계 사양서] ──┐
                   ├─→ [사양서 매칭 에이전트] ──→ [매칭 결과 DB]
[검증 테스트 사양서] ──┘                           │
                                                 │
[담당자 정보 DB] ─────────────────────────────────┼─→ [자동 알림 시스템]
                                                 │     │
[과거 매칭 이력] ─────────────────────────────────┘     │
                                                       ▼
                                              ┌─────────────────┐
                                              │   알림 전송     │
                                              ├─────────────────┤
                                              │ • 이메일       │
                                              │ • 사내 메신저  │
                                              │ • 웹 페이지    │
                                              └─────────────────┘
```

### 2.2 사양서 매칭 에이전트

사양서 매칭 에이전트는 RTL 설계 사양서와 검증 테스트 사양서를 분석하여 의미적 상관관계와 기능적 매핑을 설정한다. 에이전트는 다음과 같은 핵심 기능을 수행한다:

**의미적 분석**: 자연어 처리 기술을 활용하여 사양서 문서의 의도와 목적을 파악하고, 기능 블록과 인터페이스 정의를 식별한다.

**환경 적응**: 검증자별로 커스터마이징된 환경의 특성을 학습하여 표준 사양서와의 연결점을 찾는다.

**신뢰도 평가**: 각 매칭 결과에 대해 신뢰도 점수를 제공하여 사용자가 결과의 품질을 판단할 수 있도록 한다.

### 2.3 자동 알림 시스템

매칭 결과가 생성되면 시스템은 RTL 사양서와 검증 테스트 데이터에 기록된 담당자 정보를 자동으로 추출하여 관련 인원에게 알림을 전송한다.

**담당자 식별**: RTL 설계 사양서의 작성자/담당자와 검증 테스트 케이스의 담당자 정보를 매칭 결과와 연결하여 알림 대상을 결정한다.

**다채널 알림**: 이메일, 사내 메신저, 웹 페이지를 통해 매칭 결과와 관련 정보를 즉시 전달한다.

**상황별 알림**: 매칭 신뢰도와 문제 심각도에 따라 알림 내용과 우선순위를 조정한다.

### 2.4 적응적 매칭 전략

시스템은 검증 케이스의 복잡도에 따라 서로 다른 매칭 전략을 적용한다:

- **완벽 매칭 가능**: 직접적인 키워드 및 구조 매칭
- **부분 매칭 가능**: 의미적 유사도 분석 및 컨텍스트 추론
- **매칭 매우 어려운**: 과거 이력 기반 패턴 학습 및 전문가 지식 활용

## 3. 구현 접근법

### 3.1 데이터 통합 및 관리

검증 태스크의 복잡도와 매칭 난이도에 따라 데이터를 체계적으로 분류하고, 각 데이터에 연결된 담당자 정보를 구조화하여 관리한다. 이를 통해 매칭 결과와 담당자 알림을 효율적으로 연결할 수 있다.

### 3.2 에이전트 구현

사양서 매칭 에이전트는 대형 언어 모델과 도메인별 프롬프팅 전략을 활용한다. 데이터 분류에 따른 차별화된 처리 로직을 구현하여 각 복잡도 수준에서 최적의 성능을 달성한다.

### 3.3 통합 워크플로 자동화

매칭 완료 시점에서 자동으로 담당자 정보를 추출하고 알림을 생성하는 완전 자동화된 워크플로를 구현하였다. 이를 통해 매칭부터 담당자 알림까지의 전 과정이 끊김없이 진행된다.

## 4. 실험 및 결과

### 4.1 실험 설정

실제 SoC 프로젝트를 대상으로 실험을 수행하였다. 전체 30개 항목 중 4개 항목을 선정하여 테스트를 진행하였으며, 하나의 SoC 과제에서 특정 버전 단계만을 대상으로 우선 실험하였다. 해당 버전에서는 4개 항목에 대해 약 15만건 이상의 검증 태스크가 존재하였다.

### 4.2 데이터 분류 및 특성 분석

15만건의 검증 태스크를 매칭 복잡도에 따라 다음과 같이 분류하였다:

- **A그룹 (7만건, 46.7%)**: 완벽 매칭 가능 데이터 - 표준 검증 환경, 명확한 사양서 연결
- **B그룹 (5만건, 33.3%)**: 부분 매칭 가능 데이터 - 일부 커스터마이징된 환경, 중간 복잡도
- **C그룹 (3만건, 20.0%)**: 매칭 매우 어려운 데이터 - 고도로 커스터마이징된 환경, 불명확한 사양서 연결

### 4.3 사양서 매칭 및 알림 성능

**표 1: 데이터 분류별 매칭 정확도 및 전체 프로세스 시간**

| 데이터 분류 | 케이스 수 | AI 매칭 정확도 | 수동 방식 총 시간 | AI 방식 총 시간 | 시간 단축 효과 |
|-------------|-----------|----------------|------------------|----------------|---------------|
| A그룹 (완벽 매칭) | 70,000건 | 98.5% | 1.2시간 | 0.2시간 | 83% 단축 |
| B그룹 (부분 매칭) | 50,000건 | 87.3% | 5.1시간 | 1.5시간 | 71% 단축 |
| C그룹 (매우 어려움) | 30,000건 | 62.1% | 30.2시간 | 4.8시간 | 84% 단축 |
| **전체 평균** | **150,000건** | **85.2%** | **9.8시간** | **1.6시간** | **84% 단축** |

*총 시간: 사양서 매칭 + 담당자 식별 + 알림 전송까지 포함*

### 4.4 자동 알림 시스템 성과

**알림 전송 성공률**: 99.7% (15만건 중 149,550건 성공)
**평균 알림 전송 시간**: 매칭 완료 후 3분 이내
**담당자 식별 정확도**: 96.8% (RTL 사양서 및 검증 테스트 데이터의 담당자 정보 기반)

### 4.5 핵심 성과 분석

**A그룹 (완벽 매칭 가능)**: 98.5%의 높은 정확도로 거의 완벽한 자동화 달성. 매칭부터 담당자 알림까지 평균 12분 이내 완료.

**B그룹 (부분 매칭 가능)**: 87.3%의 정확도로 대부분의 케이스에서 유용한 결과 제공. 자동 알림을 통해 신속한 후속 조치 가능.

**C그룹 (매칭 매우 어려운)**: 62.1%의 정확도로 1차 스크리닝 역할 수행. 불완전한 매칭 결과라도 관련 담당자들에게 즉시 전달되어 신속한 전문가 개입 유도.

### 4.6 실제 적용 효과

기존 수동 프로세스에서는 사양서 매칭 후 적절한 담당자를 찾아 연락하는 과정에서 추가로 0.5-2시간이 소요되었으나, 제안 시스템은 매칭과 동시에 자동 알림을 통해 이 시간을 완전히 제거하였다. 특히 복잡한 케이스에서도 관련 담당자들이 동시에 알림을 받아 협력적 문제 해결이 가능해졌다.

### 4.7 확장성 전망

현재 4개 항목, 단일 버전에서 얻은 결과를 전체 시스템으로 확장할 경우:

**보수적 효과 추정**:
- 실제 적용 가능 범위: 전체의 약 40-60% (활성 항목/버전)
- 월 예상 처리 케이스: 약 60-90만건
- 예상 시간 절약: 월 400-600시간 
- 연간 엔지니어링 시간 절약: 약 4,800-7,200시간
- 커뮤니케이션 지연 시간 제거: 월 150-250시간 추가 절약

## 5. 향후 연구

### 5.1 매칭 성능 개선

C그룹의 정확도 개선을 위한 고급 컨텍스트 분석 기법과 도메인별 특화 모델 개발이 필요하다. 또한 사용자 피드백을 통한 지속적 학습 메커니즘 구현을 계획하고 있다.

### 5.2 지능형 담당자 할당

현재는 기존 담당자 정보를 활용한 알림 시스템이지만, 향후 연구에서는 AI 에이전트가 문제의 특성과 팀원들의 전문성을 분석하여 최적의 담당자를 지능적으로 할당하는 시스템을 개발할 예정이다.

### 5.3 다양한 검증 환경으로의 확장

현재 RTL 검증에 특화된 시스템을 다른 하드웨어 검증 영역으로 확장하는 연구를 진행할 계획이다.

## 6. 결론

본 논문은 SoC RTL 검증 워크플로에서 사양서 매칭부터 담당자 알림까지의 전 과정을 자동화하는 AI 에이전트 기반 통합 시스템을 제시하고 실제 데이터를 통한 검증을 수행하였다. 데이터 복잡도별 차별화된 접근을 통해 전체 평균 85.2%의 매칭 정확도와 84%의 시간 단축 효과를 달성하였으며, 99.7%의 자동 알림 성공률을 통해 신속한 문제 해결 체계를 구축하였다.

특히 기존에 해결하기 어려웠던 복잡한 케이스에서도 자동화된 워크플로를 통해 관련 담당자들이 즉시 협력할 수 있는 환경을 제공함으로써, 전체 검증 프로세스의 효율성을 크게 향상시켰다. 이는 검증 워크플로의 디지털 전환을 위한 실용적이고 확장 가능한 솔루션임을 입증한다.
